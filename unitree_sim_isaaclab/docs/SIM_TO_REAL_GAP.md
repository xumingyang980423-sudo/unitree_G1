# G1 Inspire Teleop 场景与 Sim-to-Real 差异说明

> 当前文档总结现有 `teleop/` 场景方案与真实机器人抓取之间的差异，并给出后续做 sim-to-real 迁移时需要补齐的工作。

---

## 1. 当前场景定位

当前 `teleop/teleop_grasp.py` 场景主要目标是：

- 在 Isaac Lab 仿真中，通过键盘遥操作 G1 右机械臂 + Inspire 灵巧手抓取红色圆柱体。
- 为后续演示数据采集、模仿学习、策略原型验证提供一个稳定可用的仿真环境。
- 优先保证仿真内抓取成功率、操作稳定性、按键可控性。

因此，目前场景更接近：

```text
遥操作 / 模仿学习原型环境
```

而不是严格的真实机器人数字孪生。

---

## 2. 当前方案中做过的仿真调参

为了让圆柱体在仿真中稳定抓取，当前场景做过多处工程调参：

| 类别 | 当前做法 | 目的 |
|------|----------|------|
| 物体尺寸 | 圆柱半径、高度按 Inspire 手抓取几何调过 | 让手指更容易包裹 |
| 物体摩擦 | 提高静摩擦 / 动摩擦 | 抬升时减少滑落 |
| 物体阻尼 | 提高线阻尼、角阻尼，限制最大速度 | 减少物体抖动和飞出 |
| 手指 PD | 提高手指刚度、阻尼、effort limit | 增强抓取支撑力 |
| 手指闭合 | 调整 index/middle/ring/pinky 的 scale 和 mimic | 减少穿模、改善包裹 |
| 拇指闭合 | 调整 yaw/pitch、tip mimic | 减少拇指穿模并增强支撑 |
| 抓取锁定 | 达到闭合阈值后锁定 finger/pinch | 移动机械臂时不因距离波动松手 |
| 工作空间限制 | 对右腕目标做边界钳位 | 避免右臂靠近身体中线导致穿模 |

这些调参对仿真稳定性有帮助，但不一定等价于真实硬件特性。

---

## 3. 与真实场景的主要差异

### 3.1 手指执行器差异

仿真里使用的是关节 position target + PD actuator，且刚度、阻尼、effort limit 是人为调高的。

真实 Inspire 手上会存在：

- 电机力矩上限
- 电流限制
- 控制带宽限制
- 通信延迟
- 齿隙 / 回差
- 关节摩擦
- 指腹软材料形变
- 硬件限位和安全保护

仿真中这些因素没有被准确建模。

---

### 3.2 手部几何与碰撞体差异

当前 USD 模型中：

- 无名指 / 小拇指名义行程较小。
- 食指 / 中指容易出现穿模，需要人为降低 scale 和 mimic。
- 拇指支撑位置和穿模问题也需要手动调参。

这说明仿真中的：

- 视觉模型
- 碰撞体
- 关节轴
- mimic 关系
- 关节限位

可能与真实 Inspire 手不完全一致。

真实迁移时，如果真实手指接触点、指腹形状或关节行程不同，策略可能会出现接触偏差。

---

### 3.3 接触与摩擦差异

当前圆柱体摩擦被调得较高：

- 静摩擦提高
- 动摩擦提高
- 物体阻尼提高

这会让仿真里的圆柱体更不容易滑落。

真实场景中摩擦取决于：

- 指腹材料
- 圆柱材质
- 表面粗糙度
- 是否有灰尘 / 油污
- 接触面积
- 法向夹持力
- 物体重心和质量分布

因此，仿真中抓稳不代表真实场景一定抓稳。

---

### 3.4 物体参数差异

当前圆柱体尺寸和质量是为了仿真抓取成功率调过的。

真实任务中可能不同：

- 半径
- 高度
- 质量
- 重心
- 表面材质
- 是否有倒角
- 是否有弹性或轻微变形

如果真实圆柱和仿真圆柱不同，策略的接触位置和夹持力都会受到影响。

---

### 3.5 机械臂控制差异

当前遥操作控制链路：

```text
键盘输入 → 末端位姿目标 → Pink IK → 右臂关节 position target
```

真实机器人控制链路通常还包括：

- 关节速度限制
- 关节加速度限制
- 轨迹插值
- 通信延迟
- 控制频率限制
- 力矩 / 电流限制
- 安全碰撞检测
- 急停和限位保护

仿真中的机械臂通常更“听话”，真实机械臂不一定能以同样方式跟踪目标。

---

### 3.6 观测差异

模仿学习训练时，如果直接使用仿真真值，例如：

- 圆柱体精确位姿
- 末端精确位姿
- 精确关节状态
- 抓取成功状态

真实系统中不一定能直接得到。

真实系统可能需要：

- 相机识别圆柱位姿
- 相机到机器人坐标系标定
- 位姿滤波
- 遮挡处理
- 时间同步
- 传感器噪声处理

如果训练输入是真值，但真实部署只能用视觉估计，策略会存在明显分布差异。

---

## 4. 直接 Sim-to-Real 的主要风险

| 风险 | 表现 |
|------|------|
| 抓取力不足 | 仿真能夹住，真机夹不住或滑落 |
| 抓取力过大 | 真机可能损坏物体或手指电机过载 |
| 摩擦不匹配 | 圆柱体真实材质更滑，抬升时掉落 |
| 接触点偏差 | 指尖在真实中没有落到仿真学到的位置 |
| 延迟影响 | 策略动作在真机上滞后，导致错过接触时机 |
| 关节限位差异 | 仿真可达姿态真机不可达或不安全 |
| 自碰撞风险 | 真机必须严格限制手臂靠近机身 |
| 观测落差 | 仿真真值策略无法直接用真实视觉替代 |

---

## 5. 如果未来要做 Sim-to-Real，需要怎么做

### 5.1 先明确真实可用观测

不要先训练依赖仿真真值的策略，而应先确认真实系统可以稳定获得哪些输入。

建议第一版真实观测：

- 右臂关节角
- 右手关节角
- 末端位姿或通过正运动学计算的腕部位姿
- 相机估计的圆柱体位姿
- 抓取阶段状态（是否已闭合、是否正在抬升）

如果真实系统暂时没有稳定视觉位姿，可以先做半自动流程，而不是端到端策略。

---

### 5.2 让仿真动作空间接近真机

当前仿真动作较理想化。迁移前建议加入：

- 动作限速
- 动作低通滤波
- 控制延迟
- 关节速度 / 加速度限制
- 手指闭合速度限制
- 力矩或 effort 饱和
- 执行误差噪声

目标是让仿真动作执行效果更像真实机器人。

---

### 5.3 重新校准手部模型

需要对真实 Inspire 手做标定：

- 每个关节的真实可动范围
- command 到实际角度的映射
- mimic / 耦合关系
- 指尖接触位置
- 拇指可达区域
- 最大安全夹持力

然后回写到仿真参数中，避免继续依赖“看起来能抓住”的手动调参。

---

### 5.4 做摩擦和物体域随机化

训练时不要固定一个理想物体参数。

建议随机化：

- 圆柱半径
- 圆柱高度
- 圆柱质量
- 圆柱摩擦
- 圆柱初始位姿
- 指腹摩擦
- 手指 PD 刚度 / 阻尼
- 控制延迟
- 观测噪声

这样策略不会只适配单一仿真设定。

---

### 5.5 采用分阶段策略，不要一步到位上真机闭环

推荐迁移顺序：

```text
阶段 1：仿真遥操作数据采集
阶段 2：仿真 BC 训练，验证抓取 + 抬升
阶段 3：加入噪声、延迟、随机化，提升鲁棒性
阶段 4：真机半自动执行（策略给目标，底层安全控制器执行）
阶段 5：小范围真机验证，只允许低速、短距离动作
阶段 6：逐步扩大动作范围，引入 DAgger 人工纠正
```

---

### 5.6 真机必须有安全约束

真实部署前至少需要：

- 右臂工作空间边界
- 关节限位
- 速度限位
- 加速度限位
- 急停
- 机身自碰撞保护
- 手指最大夹持力限制
- 掉落检测或安全放开策略

仿真中现在的 `_clamp_wrist_target()` 只能作为工作空间保护的原型，不能直接等价于真机安全系统。

---

## 6. 建议的近期路线

### 近期目标：先做仿真内模仿学习原型

当前场景适合继续做：

1. 键盘遥操作采集演示数据。
2. 训练低维状态输入的 Behavior Cloning 策略。
3. 在仿真中验证自动接近、抓取、抬升。
4. 统计成功率、滑落率、穿模情况。

这一阶段不建议立即追求真实迁移。

---

### 中期目标：增强仿真真实性

在 BC 跑通后，再补：

- 真实可用观测接口
- 控制延迟
- 动作限速
- 摩擦随机化
- 物体尺寸随机化
- 手指参数随机化

---

### 后期目标：小范围真机验证

先让策略只输出高层目标，例如：

- 目标腕部位姿
- 抓取阶段切换
- 手指闭合比例

底层仍由安全控制器执行，避免策略直接控制所有关节。

---

## 7. 结论

当前场景可以作为：

```text
遥操作调试环境 + 演示数据采集环境 + 模仿学习原型环境
```

但它还不是严格的真实机器人数字孪生。

如果未来要做 sim-to-real，需要重点补齐：

- 手指执行器建模
- 指腹 / 圆柱接触建模
- 摩擦和物体参数随机化
- 真机可用观测
- 控制延迟和限速
- 安全约束
- 小步真机验证与 DAgger 纠正

只有这些逐步完成后，仿真训练出来的抓取策略才更有机会稳定迁移到真实 G1 + Inspire 手上。
