## 1. 目标

## 3. 每步记录什么（数据字段）

### 3.1 观测 `obs_t`

- **机器人**：右臂 7 关节角/速度；右手 12 关节角/速度；右腕位姿（位置 + 四元数）
- **物体**：圆柱中心位姿、线速度/角速度（仿真可直接读）
- **可选（后续）**：腕部/俯视 RGB 或深度图（视觉策略再启用）

### 3.2 动作 `act_t`

- 与环境 `env.step(action)` **一致**的动作向量（末端位姿目标 + 手指相关控制量）
- **按键事件**（可选）：WASDQE、K/J/N/R 的按下/松开，便于回放与失败分析

### 3.3 标志与元信息 `info_t` / episode meta

| 字段 | 用途 |
|------|------|
| `grasp_ok` / `lift_ok` | 与现有 `teleop_grasp_detect` 对齐 |
| `grasp_locked` | 手指闭合锁定时刻（移动手臂时防松手） |
| `slip` / `drop` | 脱手、掉落事件 |
| `success` | 本回合是否达到「抓住并稳定抬升」 |
| `fail_reason` | 未对齐、太远按 K、滑落、穿模等 |
| `seed` / `object_init_pose` | 复现与随机化 |

### 3.4 采样频率

- 与控制循环对齐，建议 **20 Hz 或 30 Hz**（固定 `dt`，便于训练）
- 每回合长度建议 **8–20 秒**（过长噪声多、标注成本高）

---

## 4. 采集协议（决定数据上限）

### 4.1 标准单回合流程

1. `R` 或环境 reset  
2. 右臂移动到圆柱右侧合适接触区（参考 `[zone] ideal`）  
3. `K` 侧向抓取（必要时 `J` 加力）  
4. 稳定抬升约 **1–2 秒**（WASDQE 微调）  
5. 结束：放下或标记成功并进入下一回合  

### 4.2 第一版规模建议

| 项 | 建议 |
|----|------|
| Episode 数量 | **150–300 条** |
| 成功率目标 | **> 70%**（在固定/小幅随机化场景下） |
| 成功 vs 失败 | **都保存**；训练时可对成功样本过采样 |
| 参数冻结 | 采集期间**不要**改手指 PD、闭合 scale、摩擦等，否则需新开 `dataset_v2` |

### 4.3 多样化（第二版数据再加）

- 圆柱 XY 小范围随机、高度微偏  
- 摩擦/质量小幅随机  
- 初始手臂姿态轻微扰动

## 6. 模仿学习训练策略（由简到强）

### 6.1 Behavior Cloning（BC）— 第一版

- **输入**：低维状态（关节 + 物体位姿 + 可选接触区指标）  
- **输出**：连续动作（与遥操作 `action` 同维）  
- **网络**：MLP，2–3 层  
- **损失**：动作 MSE（或 Huber）  
- **优点**：实现快，易暴露数据分布问题  

### 6.2 时序策略 — 第二版

- 输入最近 **5–10 帧** 状态，或小型 GRU / Transformer  
- 改善「抬升过程中手指反应滞后、脱手」  

### 6.3 DAgger — 第三版（推荐）

- 策略自主执行 → 失败时人工接管纠正 → 合并进数据集  
- 缓解「训练分布 vs 部署分布」不一致

## 11. FAQ：是不是要自己操作 150 次？

**是的，演示学习（Imitation Learning）的第一版数据通常来自你的人类操作**——每一条 **episode** = 你从 reset 到「抓稳/抬升/结束」的**一整段操作录像（状态+动作序列）**，不是按一次按键算一条。

可以这样理解工作量：

| 概念 | 含义 |
|------|------|
| 1 episode | 约 8–20 秒的一整次尝试（成功或失败都算一条） |
| 150–300 episode | 大约 **150–300 次完整尝试**，不是 150 次按键 |
| >70% 成功率 | 其中约 **105–210 条** 应是「成功演示」，其余失败样本仍有价值（边界情况） |

**实际耗时粗算**（熟练后）：

- 单次约 15 s 操作 + 5 s reset → 约 20 s/条  
- 200 条 ≈ 4000 s ≈ **1.1 小时纯操作**（不含休息、失败重试）  
- 建议分 **多天、每次 20–40 条**，避免疲劳导致动作质量下降  

**减轻负担的做法**（后续实现录制功能时可用）：

- 一键 `R` 自动 reset + 自动开始下一条录制  
- 成功/失败快捷键打标签，减少事后筛选  
- 先录 30 条练流程，再批量录  
- 第二版用 **DAgger**：策略跑一段、你只纠正失败片段，总人工量会下降  

**失败样本要不要录？**  
要。只录成功会导致策略没见过「快滑了怎么补救」；训练时对成功样本加权即可。
